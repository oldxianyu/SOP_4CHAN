🛒 订单问题
Q1 库存充足但订单提示缺货
  1. 获取订单号 → 新零售 → B2C订单查询 → 查看订单日志，确认是否合并订单（合并订单下账后 ERP 无法显示原始平台单号）
  2. 获取商品机构和商品编码 → 新零售 → 商品 → 库存机构查询 → 查看可用库存
  3. 如可用库存为 0 → 点击"库存详情"查看是否入库或未推送库存
Q2 订单流水无法查询
问清下单时间及订单号 → B2C订单查询 → 查订单日志看是否合并订单。合并订单的流水会生成新单号，原平台单号无法直接查到。若无销售单号，查 OMS 订单查询动态报表，看备注报错信息。
Q3 订单一直显示"待审核"
联系开发刷新处理，或直接重启接口平台。
Q4 美团订单无法同步到系统
  1. 查平台套餐是否到期：https://middle.hydee.cn/operation/user.html#/login
  2. 新零售 → 设置 → 网店管理 → 店铺管理 → 确认店铺 ID 绑定到正确机构
  3. 如店铺名称可查但店铺 ID 错误 → 查授权绑定（参考"自研平台新增门店绑定操作步骤"）
Q5 订单超卖原因排查
  1. 获取问题订单号 → 查订单日志，确认锁库存是否成功（成功 = 下单时 ERP 有库存）
  2. 商品 → 第三方商品 → 查看商品下载对码时间，是否晚于订单下单时间
  3. 商品 → 库存价格查询 → 输入机构和商品编码，查下单时刻实际库存
Q6 快递信息不下发到 ERP / 面单修改
  • 数据表 t_b2c_sale_h 和 t_b2c_sale_d 中 opertype=3 时才显示快递单号
  • expressCode = 快递公司编码，expressNumber = 快递单号
  • opertype：1=接单记录，2=取消接单，3=零售下账，4=零售退货
  • 面单模板修改：新零售 → 设置 → B2C订单设置 → 模板管理 → 模板配置（改后需重新获取面单）
Q7 订单商品出库规则怎么设定
系统参数 2825 控制出库规则：
0 = 按批次先进先出 ｜ 1 = 按效期 ｜ 2 = 按库存数（多者先出）｜ 3 = 按批号（生产日期排序）
Q8 订单列表如何自定义排序
点击齿轮图标 → 找到需要的字段 → 鼠标左键按住红框图案变成十字光标 → 拖动到目标位置即可。
📦 库存问题
Q1 商品提示某批号库存为 0，但其他批号库存充足
门店数据库执行 SQL 并传输两次： update u_oms_sale_m_err set isBatchNoStock='0' where isnull(isBatchNoStock,'1')='1'
执行完 务必传输两次。如提示库存为 0 则需补充库存后重新传输。
Q2 饿了么商品同步报错 / 商品被锁定
  • 点击系统订单号查看是否有多个发货单（对接 WMS 时地址修改会新增单据）→ 线下 ERP 操作退款
  • 饿了么退款时理由选"商品缺货"→ 平台自动锁定该商品 一天内无法同步库存
Q3 门店无此商品编码，提示需拉取库存
商品 → 库存价格查询 → 点击"拉取 ERP 库存" → 勾选对应门店机构 → 输入商品编码 → 同步即可。
Q4 已下账订单如何取消下账（库存退回）
订单 → 订单查询 → 修改订单 → 修改订单状态为"已下账取消下账信息" → 输入订单号点击确定。 必须按 1→2→3→4 顺序操作，不能跳过任何一步。取消后库存自动退回。
🏷️ 商品问题
Q1 新品铺货失败，报错"药品已存在"
检查该商品在平台上是否已维护 ERP 编码。若已维护，按"商品新增对码流程"操作，不走新品铺货。
Q2 新品铺货提示"超出售卖范围 / 需补齐资质"
在平台店铺设置中补齐对应资质 → 等待平台审核通过 → 重新操作新品铺货。
Q3 新品铺货提示"请到标品库创建"
美团：外卖商家中心 → 商品管理 → 商品列表 → 新建单个商品 → 搜索 UPC/标题 → 填名称类目/UPC → 提报标品。
饿了么：一般为药品强控类目，需通过商家端 → 商品管理 → 强控标品提报 → 提交 SPU 标品信息 → 审核通过后创建。非药品类目需确认 cat3_id 正确。
Q4 订单如何换货（未下账）
订单 → 订单处理 → 销售单下账 → 找到订单点击蓝色订单号 → 勾选商品点击"换货" → 输入正确商品和金额 → 下账。
如需把一个商品换成多个，可同时勾选多个目标商品并输入各自金额。
Q5 换货时有差价怎么处理
设置 → 店铺管理 → 找到门店点击设置 → 店铺设置 → "换货是否校验总金额"改为不校验。 换货完成后必须改回，防止随意换货。
Q6 已下账订单如何换货
先按"库存 Q4"取消下账 → 库存自动退回 → 再按"商品 Q4"步骤换货 → 换货完成后在下账列表重新下账。 修改的订单不会在 ERP 产生流水。
Q7 订单中有不需要下账的商品如何剔除
换货校验改"不校验" → 销售单下账 → 点击订单号 → 勾选多个商品 → 换货 → 只输入需要保留的商品和金额 → 下账即可。
💸 退款问题
Q1 退款单无法下账
点击系统订单号查看是否有多个发货单（对接 WMS 后地址修改会新增单据）→ 多个发货单会导致线上订单和 ERP 流水号无法对应 → 线下 ERP 操作退款即可。
Q2 0 数量退款单如何处理
专项处理路径：需在退款单下账界面确认，参见"0 数量退款单专项处理"文档。
Q3 退款审核时三种选项怎么选
退货退款：库存需退回系统，下账类型选"退货退款"
仅退款：库存不退回系统，下账类型选"仅退款"
拒绝退款：驳回客户退款申请，需填写备注
Q4 退款时追溯码怎么处理
订单只有一个商品：退款单追溯码自动带出，无需手动录入。
订单有多个商品：退款单下账 → 点击订单商品 → 手动录入追溯码 → 下账。
🖨️ 打印问题
Q1 打印服务如何安装配置
  1. 下载 HydeePrintServer 安装包 → 安装前关闭杀毒软件
  2. 默认安装路径：D:\Program Files(HydeePrx86)\intTool
  3. 安装完成后启动打印服务 → 点击系统设置
  4. 输入商家编码（一般为商品编码对接群群名前几位数字）
  5. 输入门店编码 → 选择打印机 → 保存
Q2 如何测试打印是否正常
登录 O2O 账号 → 右上角账号 → 打印设置 → 点击"测试机构" → 选择机构 → 打印测试小票和语音播报。如有订单可直接点击"重打小票"。
Q3 如何修改小票模板 / 分配给门店
修改：登录新零售 → 设置 → 订单设置 → 小票模板设置 → 修改或新增模板。
分配门店：设置 → 门店管理 → 店铺管理 → 批量修改配置 → 勾选门店 → 订单处理设置 → 选择模板分配。
⚙️ 系统 & 接口问题
Q1 系统卡单 / 接口异常怎么办
联系开发刷新处理，或直接重启接口平台。
Q2 登录新零售提示"无权限访问"
检查账号权限分配是否正确 → 联系管理员核对组织员工权限配置（参考"组织员工权限分配"视频）。
Q3 H1/H2 ERP 相关系统设置
系统参数已设置且为联营商品时需参考相应文档。参见"H1 ERP相关设置.docx"和"H2 ERP相关设置.docx"。
📋 操作流程
Q1 骑手配送完整流程
来单 → 订单进入待拣货 → 点击拣货 → 录批号 → 选拣货员 → 完成拣货 → 骑手取货。 必须在骑手取货前完成拣货，否则需手动下账。如需代发货可在呼叫骑手前修改门店。
Q2 门店自提完整流程
来单 → 待拣货 → 拣货（录批号+选拣货员+完成拣货） → 顾客到店 → 点击"到店取货" → 选配送员工 → 确定。
Q3 员工配送完整流程
来单 → 待拣货 → 拣货（录批号+选拣货员+完成拣货） → 点击"配送出库"选配送员 → 点击"确认妥投"完成。
Q4 拣货时追溯码怎么操作
拣货复核 → 提示"待扫码" → 录批号 → 手动输追溯码或用扫码枪扫。 旧包装无追溯码：在"无码商品"输入商品数量即可正常下账。
Q5 ERP 中如何查看追溯码记录
路径一：零售流水账查询 → 右侧"追溯码"按钮。
路径二：码上放心上传记录 → 按追溯码查询（仅扫了大包装码的才会下载小包装码到码清单）。
Q6 退款审核标准流程
顾客申请退款 → 退款中界面 → 点击"退款审核" → 弹出三种选择：退货退款 / 仅退款 / 拒绝退款 → 选择后打备注 → 提交完成。
Q7 销售单已下账后如何退款
退款单下账 → 选择下账审核类型：
退货退款：库存退回系统
仅退款：库存不退回
点击下账即可完成。
Q8 新品铺货前发现商品无条形码
商品库 → 编辑 → 维护条形码。或双击 ERP 编码 → 修改 → 补充条形码。
Q9 新品铺货后关联数据未更新怎么排查
商品 → 第三方商品 → 任务进度 → 找到"新品铺货"任务记录 → 若提示失败 → 点击"下载结果"查看详细失败原因。

Q4 饿了么开发者账号如何申请
门店大于 1 个时，饿了么总管账号即为开发者账号。单体门店无法申请，需新增门店后再操作。
Q5 打印服务需要哪些编码，哪里找
商家编码：一般为商品编码对接群群名前几位数字。
门店编码：新零售 → 订单处理 → 页面括号内即为门店编码；ERP → 机构编码也可作为门店编码。
